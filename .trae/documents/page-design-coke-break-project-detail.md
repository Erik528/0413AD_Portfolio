# “COKE BREAK”项目详情页｜页面设计说明（桌面端优先）

> 目标：严格按截图布局实现详情页，并复用首页的导航、字体与背景风格；将页面拆分为可复用区块与组件，便于后续添加更多项目详情页。

## 1) Layout（栅格与响应式）
- 主容器：居中 `max-width: 1120–1200px`，左右 `padding` 随断点变化（Desktop 48px / Tablet 32px / Mobile 20px）。
- 栅格：桌面端 12 列 CSS Grid。
  - Hero 区：左侧信息（约 5 列）+ 右侧主视觉媒体（约 7 列），与截图一致呈左右分栏。
  - 内容区：按截图呈“标题/说明”编辑式排版（建议 4/8 列或 5/7 列分栏；保持与 Hero 的列宽体系一致）。
- 间距：区块垂直间距使用 24/32/48px 三档；区块内元素 8/12/16px。
- 响应式：
  - <= 768px：Hero 改为上下堆叠（标题与摘要在上，主视觉在下），项目信息列表下移。
  - 文本行宽：正文最大行宽建议 60–75 字符（避免过宽）。

## 2) Meta Information（SEO/分享）
- Title：`COKE BREAK | Work`
- Description：`COKE BREAK 项目详情：摘要 + 你的核心亮点（不超过 120 字）`
- Open Graph：
  - `og:title` 同 Title
  - `og:description` 同 Description
  - `og:image` 使用主视觉图（与截图中的大图一致）

## 3) Global Styles（复用首页风格约束）
> 不新造全站风格；本页仅“引用/复用”首页已存在的 token 与组件。
- 背景：复用首页背景（同色值/同纹理/同噪点或渐变）。
- 字体：复用首页字体家族与字号体系。
- 颜色：
  - 主文本：与首页正文一致
  - 次级文本：与首页 muted 文本一致
  - 强调色：复用首页 accent（截图中用于标题或小标识的强调）
- 按钮/链接：
  - 链接默认下划线或 hover 下划线（与首页一致）
  - hover：仅改变颜色/透明度，不改变布局（避免抖动）

## 4) Page Structure（与截图一致的区块结构）
自上而下：
1. 全站顶部导航（复用首页）
2. 页面标题区（含返回入口）
3. Hero 双栏（左信息 + 右主视觉）
4. 项目信息列表（与截图中信息块的样式一致：列表或小卡排布）
5. 内容分区（编辑式排版，多段落/小标题）
6. 底部“上一个/下一个项目”导航 + Footer（如首页已有）

## 5) Sections & Components（区块与可复用组件定义）

### 5.1 复用全站组件（必须与首页一致）
1) `SiteNav`
- 位置：页面最上方
- 内容：Logo/站点名 + 主导航项
- 行为：当前路由高亮；滚动时保持与首页一致（固定/非固定都按首页）

2) `SiteBackground`
- 说明：页面根容器直接复用首页背景实现方式（同 class/token）

3) `Typography`（全局）
- 说明：本页仅使用首页现有的 `h1/h2/body/small` 级别，不新增字号。

### 5.2 详情页专属区块（建议做成可复用的 Project 级组件）

A) `BackLink`
- 位置：标题区最上方（截图中通常在标题上方或左上）
- 文案：`← Back` / `← 返回作品`
- 交互：hover 强调；点击返回列表或首页

B) `ProjectHeader`
- 内容：
  - `ProjectTitle`：大标题 `COKE BREAK`（与截图字号/字重一致，单行优先）
  - `ProjectMeta`：副标题/标签（如“Branding / Packaging”等，按你的内容源）
  - `ProjectSummary`：1 段摘要文案（2–4 行，行高略大于正文）

C) `ProjectHero`
- 布局：12 列栅格
  - 左：`ProjectHeader`
  - 右：`MediaFrame`

D) `MediaFrame`（关键：截图中的大图容器）
- 视觉：
  - 固定圆角（如 16–24px，按首页卡片风格）
  - 可选轻阴影或 1px 边框（与首页卡片一致）
  - 保持媒体比例稳定（建议 `aspect-ratio: 4/3` 或与截图一致的比例）
- 内容：默认 1 张主图；后续可扩展为视频/轮播但不改变布局占位
- 行为：点击放大（轻量方案：打开原图新标签）

E) `ProjectFacts`
- 目的：承载“年份/角色/周期/工具/交付物”等信息
- 结构：
  - 桌面端：2 列或 3 列小列表（按截图的信息块密度）
  - 每项：`FactItem(label, value)`
- 文案规则：label 短、value 可换行；对齐方式与截图一致（常见为左对齐）

F) `ProjectSection`（可复用：每个内容分区一个组件实例）
- 输入：`title` + `body`（支持多段落/项目符号）
- 排版：
  - 左列：Section 标题（更醒目、与正文分层）
  - 右列：正文内容（段落间距明确）
- 建议分区（按你的内容有多少就渲染多少，不强制）：背景/挑战/方案/过程/结果

G) `ProjectPager`
- 位置：正文结束后、Footer 之前
- 内容：
  - 上一个项目（左）
  - 下一个项目（右）
  - 可选：中间“Back to Work”
- 交互：大点击区域；hover 显示强调色

## 6) Interaction States（必要交互态）
- 链接/按钮：hover 颜色或下划线（与首页一致）
- 媒体：hover 可轻微放大或提高对比度（可选；不破坏截图的克制感）
- 锚点（可选）：若你要在右侧加目录，则滚动高亮当前分区（仅在你确有该需求时启用）

## 7) Content Checklist（落地时需要的最小内容）
- 标题：COKE BREAK
- 摘要：1 段
- 主图：1 张（与截图一致位置与尺寸）
- Facts：至少 4 项（年份/角色/周期/工具…）
- 内容分区：至少 3 个（保证页面信息密度与截图相近）
- 上下个项目 slug（若暂无，可先隐藏 `ProjectPager`）
